## 2. ИИ-консультант в мессенджере MAX

### Что это
Чат-бот в мессенджере MAX, который отвечает клиентам на вопросы об ассортименте, ценах и доставке. Работает на базе Google Gemini AI.

### Ссылка на бота
- Мобильный (правильная ссылка): `https://max.ru/id590300930576_bot`
- Веб-версия: `https://web.max.ru/23965227`
- ⚠️ Ссылка с `web.` работает только на компьютере. Для мобильных — только без `web.`

### Технический стек
| Компонент | Что используется |
|-----------|-----------------|
| Мессенджер | MAX (mail.ru) |
| AI-модель | Google Gemini 2.5 Flash (`gemini-2.5-flash`) |
| Сервер | Flask (Python) |
| Хостинг | **Hostinger VPS (KVM 2)** — работает 24/7, не засыпает |
| Адрес сервера | `srv1495789.hstgr.cloud` (HTTPS) / IP: `72.60.131.145` |
| Обратный прокси | nginx + Let's Encrypt SSL (до июня 2026, продлевается авто) |
| Вебхук MAX | `https://srv1495789.hstgr.cloud/webhook` |
| Хранение кода | GitHub: `Kaidelena/znatniysad-bot` |

### Как обновить знания бота
1. Открыть файл `main.py` в репозитории на GitHub
2. Найти переменную `SYSTEM_PROMPT`
3. Добавить новую информацию внутрь
4. Загрузить файл (Upload → перетащить → Commit)
5. Зайти на сервер через SSH и перезапустить бота:
   ```
   ssh root@72.60.131.145
   systemctl restart znatniysad
   ```

### Важные технические детали
- Модели `gemini-2.0-*` **устарели** для новых пользователей с 6 марта 2026. Использовать только `gemini-2.5-flash`
- Переменные окружения (`GEMINI_API_KEY`, `MAX_BOT_TOKEN`) хранятся в файле `/root/znatniysad-bot/.env` на сервере
- Бот управляется через systemd: `systemctl start/stop/restart/status znatniysad`
- При перезагрузке сервера бот стартует автоматически (enabled в systemd)
- nginx проксирует HTTPS → Flask на порту 5000
- SSL сертификат от Let's Encrypt, продлевается автоматически через certbot

### 25–26 марта 2026 — Переезд с Render.com на Hostinger VPS
**Зачем:** Render.com (бесплатный план) "засыпал" при неактивности — клиенты ждали ответа 50+ секунд. VPS уже был оплачен — смысла платить за Render не было.

**Что сделали пошагово:**
1. Подключились к VPS через SSH (`ssh root@72.60.131.145`)
2. Создали папку `/root/znatniysad-bot`
3. Скачали `main.py` с GitHub командой `curl`
4. Установили Python-зависимости в виртуальное окружение (`venv`)
5. Создали файл `.env` с ключами `GEMINI_API_KEY` и `MAX_BOT_TOKEN`
6. Создали systemd-службу `/etc/systemd/system/znatniysad.service`
7. Установили nginx и получили SSL-сертификат через certbot (Let's Encrypt)
8. Настроили nginx как обратный прокси (HTTPS → порт 5000)
9. Обновили вебхук в MAX через API: `https://srv1495789.hstgr.cloud/webhook`

**Результат:** Бот отвечает мгновенно, работает 24/7, HTTPS настроен.
